## UI Overhaul + File Browser + GitHub Integration Plan

### What needs to change

The current UI works but lacks:
1. **No file upload** — "Upload Files" source does nothing (backend has no upload endpoint)
2. **No file browser** — users can't browse local files/folders
3. **No GitHub URL flow** — the GitHub card doesn't accept a repo URL
4. **Dated visuals** — basic CSS, no drag-and-drop, no animations

### Backend Changes (2 new endpoints)

**1. `POST /v1/documents:upload`** — Multipart file upload endpoint
- Accepts `multipart/form-data` with multiple files + `collection_name`
- Saves files to `./data/uploads/{job-id}/`
- Creates an IndexJob that ingests the uploaded files
- Returns `{ job_id, collection_id, file_count }` (JSON)

**2. `POST /v1/documents:github-lookup`** — Parse GitHub URL
- Accepts `{ url: "https://github.com/owner/repo" }`
- Extracts owner/repo, calls GitHub API to verify
- Returns `{ owner, repo, description, stars, default_branch }` or falls back to just the path

These are added to `api/routers/documents.py` and registered in `api/server.py`.
